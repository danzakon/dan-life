# Technical Guides

Step-by-step guides for setting up tools, infrastructure, and development environments.

---

## Purpose

This directory contains technical documentation written for personal reference. Guides should be practical, focused, and reusable.

---

## Writing Style

- Start with a clear overview of what the guide accomplishes
- Use diagrams to illustrate architecture and data flow
- Prefer tables for commands, options, and comparisons
- Include troubleshooting sections for common issues
- Add checkboxes for multi-step setups that can be tracked

---

## File Naming

```
{topic}-{subtopic}.md

Examples:
hetzner-remote-dev-setup.md
docker-compose-basics.md
neovim-lsp-config.md
```
