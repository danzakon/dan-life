## 20260309-RS-001 — The MCP Token Tax: CLI vs MCP Token Economics

**Source:** research
**Raw file:** `research/reports/20260309-mcp-vs-cli-token-economics.md`
**Status:** raw

### Summary

Quantitative research report on the token economics of MCP (Model Context Protocol) vs CLI-based tool calling for AI agents. The GitHub MCP server burns ~55,000 tokens on tool schemas alone. A typical 6-server production setup wastes 60,000–150,000 tokens — 30–75% of a 200K context window — before the agent does any work. CLI achieves the same capabilities at 92–98% lower token cost. Solutions are emerging (Anthropic's Tool Search, Cloudflare's Code Mode, CLIHub, mcp2cli) but the MCP spec itself needs to adopt lazy loading as a first-class primitive. Cost difference at scale: $8,100/month (naive MCP) vs $45/month (CLI) for 30K requests.

## 20260309-RS-003 — MCP vs CLI for AI Agents: The Enterprise Case That CLI Purists Keep Ignoring

**Source:** research
**Raw file:** `research/reports/20260309-mcp-vs-cli-enterprise-defense.md`
**Status:** raw

### Summary

Deep investigation into where MCP genuinely outperforms CLI for AI agents — enterprise auth (OAuth 2.1 with delegated authority and dynamic client registration), non-developer users who never touch terminals, SaaS tools that ship MCP servers but no CLIs (HubSpot, Salesforce, Notion, Asana), stateful sessions with bidirectional streaming, and governance/compliance infrastructure. MCP has 30+ CVEs and real breaches, but its fine-grained authorization model provides security tooling CLI can't match. The token overhead problem is being solved via progressive disclosure (85-100x reduction) and Claude Code's automatic MCP Tool Search. Production teams are converging on a hybrid: CLI for dev tools, MCP for SaaS and enterprise.

### Content Angles

- **Thread**: 8 things CLIs can't do that MCP handles natively — OAuth delegation, multi-tenant auth, non-developer users, stateful sessions, SaaS without CLIs, audit trails, tool filtering, governance policies. With sources and data.
- **Article**: Full editorial — "The Enterprise Case for MCP: Why CLI-Only Agent Architectures Hit a Ceiling at 30% of Business Use Cases"
- **Hot take**: The MCP vs CLI debate is a developer echo chamber. 99% of knowledge workers have never opened a terminal. MCP isn't competing with CLI — it's serving the audience CLI can't reach.
- **Post**: "HubSpot has an MCP server. Salesforce has an MCP server. Notion, Asana, Google Ads, Shopify — all MCP servers. None of them have CLIs. None of them ever will. The 'just use CLI' crowd is building for 1% of the workforce."

## 20260309-RS-004 — The CLI Advantage: Why AI Models Are Fluent in Shell but Fumble MCP

**Source:** research
**Raw file:** `research/reports/20260309-cli-vs-mcp-training-data-advantage.md`
**Status:** raw

### Summary

Deep investigation into the training data asymmetry that explains why LLMs are dramatically better at CLI tools than MCP schemas. Shell/bash content saturates every major pretraining corpus (The Stack's 3.1 TB, Stack Overflow, GitHub READMEs, Dockerfiles, Makefiles) while MCP has ~15 months of existence and near-zero training representation. Terminal-Bench 2.0 shows frontier models scoring 55–65% on hard CLI tasks. Google's gws deliberately deleted its MCP server in favor of CLI. NVIDIA demonstrated RL training for novel CLI tools transfers efficiently because models already understand shell conventions. The agent-native CLI movement (CLIHub, mcp2cli, clime, gws, klaw) is the industry response to MCP's fundamental architecture problem.

### Content Angles

- **Thread**: The Training Data Chasm — why LLMs can use `git` zero-shot but fumble MCP schemas. Billions of tokens of shell content vs 15 months of MCP existence. Numbers, benchmarks, and what Google did about it.
- **Article**: Full editorial treatment — "The CLI Advantage: 50 Years of Training Data vs 15 Months of JSON Schemas"
- **Hot take**: MCP's biggest problem isn't the protocol — it's that no model has ever been trained on it. CLI commands have 50+ years of representation in every pretraining corpus on earth. MCP has zero. The models are telling us what interface they want.
- **Post**: "Andrej Karpathy: CLIs are exciting *because* they're legacy tech. Google gws: built an MCP server, tested it, deleted all 1,151 lines. OpenClaw creator: 'MCP is a crutch.' The training data doesn't lie — LLMs know bash, they don't know JSON-RPC."
