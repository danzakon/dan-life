## [ ] Dorsey on attention to detail / vibe coding trap

**Platform:** Both
**Source:** thought-bank-2026-02.md

**Twitter:**
Jack Dorsey: to have great attention to detail, limit the number of details, then make every one perfect.

This is the lesson vibe coding hasn't learned yet.

You can build a full product in an afternoon now. So everyone's instinct is to keep adding. More features, more pages, more integrations. None of them polished.

The skill that matters most right now? Knowing when to stop building and start refining.

**LinkedIn:**
Saw a clip of Jack Dorsey talking about attention to detail. His point: limit the number of details, then make each one perfect.

I keep thinking about this in the context of vibe coding.

The tools are incredible now. You can go from zero to deployed product in an afternoon. But I've noticed a pattern in what people build with them: tons of features, none of them great.

The speed of creation has become an excuse to skip refinement. Everyone's adding more when they should be cutting.

I think the defining skill of this era is going to be editorial discipline. Taking twenty features down to five and making those five feel inevitable. Testing, iterating, reducing complexity instead of adding it.

Building less, but building it right.
